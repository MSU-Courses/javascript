## JavaScript — динамически типизированный язык

JavaScript — **динамически типизированный** язык программирования.

Простыми словами, **динамическая типизация** - это когда тип данных переменной определяется автоматически на основе её
значения. Другими словами, переменная может хранить значения разных типов в разное время выполнения программы. Например,

```js
let x = 5; // Переменная x изначально содержит число
x = "Hello"; // Теперь переменная x содержит строку
x = true; // Теперь переменная x содержит логическое значение
```

В данном примере переменная `x` изменяет свой тип данных в процессе выполнения программы: сначала это число, затем
строка, и, наконец, логическое значение.

Можно заметить, что мы явно не указываем тип данных при объявлении переменной - JavaScript сам определяет тип в
зависимости от значения, с которым она была инициализирована

## Типы данных в JavaScript

Типы данных в JavaScript делятся на **примитивные** и **ссылочные (объектные)** типы данных.

Всего стандарт ECMA определяет 8 типов данных: 6 примитивных типов и 2 ссылочных типа.

> [!NOTE]
> **Примитивные типы данных** - это основные типы данных, которые представляют собой непосредственные значения, а не
> объекты. Они не имеют методов и обрабатываются непосредственно как значения. **Например**: число, строка, булево
> значение.

В данной главе мы рассмотрим только примитивные типы данных (6 типов данных). Об объектных типах мы поговорим в отдельной теме, чтобы более детально изучить их и освоить их применение.

## Числовой тип данных `number`

> [!IMPORTANT]
> Числовой тип данных (`number`) — это тип данных, представляющий числовые значения, в формате 64-битного числа двойной
> точности с плавающей запятой.

### Примеры

```js
let integerNumber = 42; // Целое число
let floatingNumber = 3.14; // Число с плавающей точкой
let scientificNotation = 6.022e23; // Число в научной записи (6.022 * 10^23)
let negativeNumber = -10; // Отрицательное число
// и так далее ...
```

### Основные особенности

* В JavaScript нет специального типа для целочисленных значений. Все числа воспринимаются как числа с плавающей точкой.
* Таким образом, любое число, вне зависимости от его формата в JavaScript, является типом данных `number`.
* В отличие от JavaScript, в других языках программирования могут существовать различные типы данных для чисел, такие
  как `Integer` (целые числа), `Float` (числа с плавающей точкой).

Аналогом числового типа данных в JavaScript в языке C++ является тип данных `double`.

### Константы

Основными константами являются:

* `Number.MAX_VALUE` — максимальное числовое значение, представимое в JavaScript.
* `Number.MIN_VALUE` — минимальное положительное числовое значение, представимое в JavaScript.

### Операции

```js
let a = 3;
let b = 5;

// сложение
a + b;

// вычитание
a - b;

// умножение
a * b;

// деление
a / b;

// возведение в степерь (с ES6)
a ** b; // `a` в степени `b`

// логические операторы
a > b;
a < b;
a === b;

// ...
```

### Специальные числовые значения

Одной из особенностей типа number в JS являются «**специальные числовые значения**»:

1. `Infinity` и `-Infinity` — математическая бесконечность. Числовое значение, которое больше любого числа.
2. `NaN` (**N**ot **a** **N**umber - не число) — появляется в результате математических операций, которые не могут быть
   выполнены или возвращают нечисловой результат (т.е. вычислительная ошибка).

```js
console.log(1 / 0); // Результат: Infinity
console.log("a" / 2); // Результат: NaN, так как операция деления не может быть выполнена с символьным значением "a"
console.log(Math.sqrt(-1)); // Результат: NaN
console.log(0 / 0); // Результат: NaN
console.log(NaN + 2) // Результат: NaN
```

> [!CAUTION]
> Исключением является `NaN ** 0` (NaN в степени 0), которое равно 1, так как любое число, включая NaN, возводимое в
> степень 0, равно 1.

## Тип данных `undefined`

> [!IMPORTANT]
> `undefined` — это примитивный тип данных, который автоматически присваивается переменным при их объявлении или
> аргументам функции, если значение не было установлено.

### Примеры

```js
let a; // a = undefined

function foo(b) {
    console.log(b);
}

foo(); // b = undefined, так как при вызове функции foo не передан аргумент, и переменной b не присвоено значение.
```

### Операции

```js
let a;

// операция сравнения
if (a === undefined) {
    console.log("a is undefined!");
}

// математические операции с undefined дают приводят к результату `NaN`
3 + undefined; // Результат: NaN
undefined + undefined; // Результат: NaN
```

## Тип данных `boolean`

> [!IMPORTANT]
> **Boolean** (Булев, Логический тип данных) — это примитивный тип данных в JavaScript, который может принимать только
> два значения: true (истина) или false (ложь). Этот тип данных часто используется для условных операций и логических
> выражений. **Например**, если у нас есть выражение `2 > 1`, то результат будет `true`, потому что это верное
> утверждение. А если мы проверяем, например, `3 < 1`, то результат будет `false`, потому что это неверное утверждение.

> [!TIP]
> В JavaScript `true` представляет собой числовое значение **1**, а `false` - **0**.

```js
Number(true); // 1
Number(false); // 0
```

### Пример

```js
let result = 8 > 2;
console.log(result); // Результат: true (результатом данного выражения будет 'правда')

// пример использования с условным оператором
let a = 3;
if (a === 3) {
    console.log("a is 3!")
}
```

### Операции

При **сложении** (+), **вычитании** (-), **делении** (/), **умножении** (*), **возведении в степень** (**) булевого
значения (`boolean`) и числа (`number`), `true`
интерпретируется как 1, а `false` как 0, поэтому операция сложения будет работать как с числами.

```js
1 + true; // 1 + 1 = 2
false + 6; // 0 + 6 = 0

// попробуйте сами выполнить это выражение в консоли и проанализировать результат
true + false;
2 ** true;
```

## Тип данных `string`

> [!IMPORTANT]
> **Строки** в JavaScript представляют собой последовательности символов, заключенные в кавычки (одинарные или двойные).
> Они могут содержать буквы, цифры, символы и пробелы.

### Примеры

```js
let str = "I am a string!";
console.log(str); // Результат: I am a string!
```

В JavaScript, строки должны быть заключены в кавычки. Можно использовать как двойные кавычки, так и одинарные.

```js
let str1 = "Hello";
let str2 = 'World';
console.log(str1 + str2);
```

Также в JavaScript существуют обратные кавычки, которые позволяют встраивать переменные внутрь строки

```js
let name = "Alex";
console.log(`Ваше имя: ${name}`);
// Результат: Ваше имя: Alex.
// Чтобы встраивать переменные в строку, используются только обратные кавычки.

// Встраивание переменных с помощью одинарных или двойчных кавычек работать не будет
console.log("Ваше имя ${name}"); // Результат: Ваше имя ${name}
```

Для более уверенного управления строками, попробуйте использовать разные типы кавычек в зависимости от ситуации.

### Операции

Строки поддерживают множество операций, таких как конкатенация, сравнение и т.д.

```js
// Конкатенация строк
"my " + "name"; // Результат: "my name"

// Сравнение строк
"myName" === "myName" //  Результат: true
```

Мы более подробно рассмотрим различные методы работы со строками позже, так как в данный момент это может усложнить
понимание.

## Тип данных `symbol`

> [!NOTE]
> Тип данных `symbol` был введен в ECMAScript 2015 (ES6).

> [!IMPORTANT]
> Тип данных `Symbol` в JavaScript представляет собой уникальное и неизменяемое значение, которое используется для
> создания уникальных идентификаторов

### Пример

```js
let smbl1 = Symbol('name');
let smbl2 = Symbol('name');

console.log(smbl1 === smbl2);
// Выражение вернет false, потому что каждый символ создает уникальное значение
// Хотя оба символа имеют одинаковое описание `name`
```

## Тип данных `bigint`

В JavaScript тип number (рассмотренный выше), не может безопасно работать с числами, больше, чем $2^{51} - 1$ (
$9007199254740991$) или меньше, чем $-2^{51} - 1$.

Для этого используется тип данных `BigInt`.

> [!IMPORTANT]
> `BigInt` — Числовой тип данных, который может представлять данные в
> формате ['длинной арифметики'](https://en.wikipedia.org/wiki/Arbitrary-precision_arithmetic). То есть простыми
> словами,
> тип данных, который может обрабатывать очень большие или очень маленькие числа.

### Примеры

```js
// Для создания переменной типа BigInt необходимо использовать суффикс 'n' или функцию BigInt.
let a = 99748375904385934584395893408054n;// символ "n" в конце означает, что это BigInt
let b = BigInt(997483759043854);
```

> [!NOTE]
> Функция `BigInt()` преобразует числа в тип `bigint`. Если число большое, то необходимо добавлять суффикс 'n'.
> Например, для BigInt(42) добавлять суффикс не требуется, так как само число 42 не является большим. Однако, для
> BigInt(
> 9978894837590438540990n) нужно использовать суффикс 'n', так как само число 9978894837590438540990 является bigint.

Одной из особенностей **BigInt** является невозможность их сложения с типом **Number**.

```js
let a = BigInt(89584985498589994989909090905498594854);
console.log(a + 8); // Ошибка, потому что JavaScript не может выполнить операцию сложения между значением типа bigint и числом типа number.
```

## Оператор `typeof`

Хотя мы не обсуждали понятие "операторов", которое, вероятно, вам известно из курса по основам программирования, и не касались операторов в JavaScript, один из них прекрасно вписывается в нашу тему и его знание поможет вам в дальнейшем разбираться с типами данных.

Вы никогда не задумывались, как проверить какой тип данных имеем переменная?

Для этого используют оператор `typeof`.

```js
let x1 = 3;
let x2 = 78798789n;
let x3 = "hello, world";

// узнать тип данных переменных
console.log(typeof x1); // number
console.log(typeof x2); // bigint
console.log(typeof x3); // string
```

Оператор typeof имеет и другой синтаксис, напоминающий вызов функции.
Но данный синтаксис встречается крайне редко.

```js
let x1 = 'hello';
let x2 = 5;
console.log(typeof(x1 + x2)) // string
```
